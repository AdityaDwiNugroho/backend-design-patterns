# Contributing

Thank you for considering contributing to Backend Design Patterns!

## Core Principles
1.  **Zero Dependencies**: We aim to implement patterns using standard libraries only.
2.  **Strict Types**: No `any`. All code must pass strict TypeScript checks.
3.  **No Fluff**: Comments should explain *why*, not *what*. Avoid verbose "tutorial style" code.
4.  **Tests**: All logic must be covered by unit tests.

## Process
1.  Fork the repo.
2.  Create a feature branch.
3.  Implement your pattern in `src/<pattern-name>`.
4.  Add tests in `src/<pattern-name>/<pattern>.test.ts`.
5.  Submit a PR.

## Style Guide
- Use Prettier for formatting.
- Follow the existing folder structure.
